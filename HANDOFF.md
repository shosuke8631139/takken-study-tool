## 📐 問題DBの構造

```javascript
// 通常問題
{id:N, cat:'業法'|'権利関係'|'法令'|'税その他', rank:'S'|'A'|'B'|'C',
 q:'問題文',
 c:['選択肢0','選択肢1','選択肢2','選択肢3'],
 a:0,          // 正解インデックス（0〜3）
 ex:'詳細解説', pt:'覚えるポイント', ez:'中学生向けかんたん説明'}

// 個数問題（type:'count' 付き）
{id:N, cat:'業法', rank:'S', type:'count',
 q:'次の記述のうち正しいものはいくつあるか。\nア．...イ．...ウ．...エ．...',
 items: [{text:'ア．...', correct:true}, {text:'イ．...', correct:false}, ...],
 c:['一つ','二つ','三つ','四つ'],
 a:2,  // 正解インデックス
 ex:'解説', pt:'ポイント', ez:'かんたん説明'}
```

**ランク基準：**
- S = 毎年ほぼ確実に出る最重要
- A = 2〜3年に1回以上の頻出
- B = 数年に1回の標準
- C = たまに出る応用

## 🔧 問題追加の手順（必ず守ること）

```
1. ]; // DB end の直前の最後の問題が }, で終わっているか確認
2. ]; // DB end の直前に新問題を挿入
3. 挿入後に PowerShell で構文チェック
```

**構文チェックコマンド：**
```powershell
$file = "C:\Users\kurro\OneDrive\Desktop\宅建紙を取るためにサポートしてくれるツール。\index.html"
$tempFile = "$env:TEMP\takken_check.js"
$lines = Get-Content $file -Encoding UTF8
$scriptStart = -1; $scriptEnd = -1
for ($i = 0; $i -lt $lines.Count; $i++) {
    if ($lines[$i] -match '^<script>' -and $scriptStart -eq -1) { $scriptStart = $i + 1 }
    if ($lines[$i] -match '^</script>' -and $scriptStart -ne -1) { $scriptEnd = $i - 1; break }
}
$jsContent = ($lines[$scriptStart..$scriptEnd]) -join "`n"
[System.IO.File]::WriteAllText($tempFile, $jsContent, [System.Text.Encoding]::UTF8)
node --check $tempFile
if ($LASTEXITCODE -eq 0) { "構文OK" } else { "構文エラーあり" }
```
